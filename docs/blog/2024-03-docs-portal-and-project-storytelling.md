# Docusaurus Portal and Project Storytelling

Published: **March 2024**

By early 2024, TraderX had accumulated meaningful technical depth. The next problem was discoverability.

We needed better answers for first-time visitors:

- What is this project?
- How do I run it?
- Which UI/runtime path should I start with?
- Where are the architectural boundaries?

Major changes in this period addressed exactly that:

- Docusaurus was introduced and integrated (`#138` / docs-website merge).
- Homepage and UX refinements improved clarity (`#129`).
- README and demo-link quality improved in follow-up patches.

This was the first strong step toward treating TraderX as a learning portal, not only a code repository.

That direction later became the foundation for visual state mapping and spec-linked documentation, which are now central to the SpecKit-era workflow.

# TraderX Learning-Path Architecture Context

## Goal

Model TraderX as progressive learning states with explicit operational and architectural deltas.

## Components

- Core services: account, trade, position, reference-data, people
- Frontend: `web-front-end`
- Data and messaging support: database, trade-feed, trade-processor
- Learning orchestration: `docs/guide`, `states`, `prompts`, `specs`

## Levels

- Level 0: local baseline orientation
- Level 1: service decomposition focus
- Level 2: containerization and local orchestration
- Level 3: service mesh controls and observability
- Level 4: contract/spec driven evolution
- Level 5: AI-agent-assisted regeneration and drift management
